This function returns the ocean loading displacement amplitude, frequency, and phase of a set of tidal constituents generated by the Bos-Scherneck website at http://www.oso.chalmers.se/~loading/. The variable nin is input as the number wanted, and the variable nout is returned as the number provided. The constituents used are stored in the arrays idd (Doodson number) and tamp (Cartwright-Edden amplitude). The actual amp and phase of each of these are determined by spline interpolation of the real and imaginary part of the admittance, as specified at a subset of the constituents.

Parameters
[in]ampinCartwright-Edden amplitude of tidal constituents (size nin).
[in]idtinDoodson number of tidal constituents (size 6xnin).
[in]phinPhase of tidal constituents
[in]ninNumber of harmonics used
[in]itmDate as integer array in UTC. The format should be: [year,day_of_year,hours,minutes,seconds].
[out]ampAmplitude due to ocean loading
[out]fFrequency due to ocean loading
[out]pPhase due to ocean loading
[out]noutNumber of harmonics returned
Returns
zero
Note
  1. The phase is determined for a time set in COMMON block /date/ in the function TDFRPH.
  2. The arrays F and P must be specified as double precision.
  3. Status: Class 1 model
Version
2009 August 19

This function performs cubic spline interpolation of a given function sampled at unequally spaced intervals. The subroutine SPLINE needs to be called beforehand to set up the array s.

Parameters
[in]yThe coordinate at which a function value is desired (Note 1)
[in]nnNumber of samples of the original function
[in]xArray containing sample coordinates x(1),x(2),...x(nn) (Note 2)
[in]sArray containing the 2nd derivatives at the sample points (Note 3)
[in]uArray containing samples of a function at the coordinates x(1),x(2),...x(nn)
[out]valThe interpolated value of the function at y
Returns
Zero on success; else 1.
Note
  1. If y falls outside the range (x(1),x(nn)), the value at the nearest endpoint of the series is used.
  2. The sequence x(1),x(2),...x(nn) must be strictly increasing.
  3. This array is found by the function SPLINE, which must be called once before beginning this interpolation.

int iers2010::hisp::tdfrph ( const int *  idood,
const int *  itm,
double &  freq,
double &  phase 
)

This subroutine returns the frequency and phase of a tidal constituent when its Doodson number is given as input.

Parameters
[in]idoodDoodson number of a tidal constituent
[in]itmDate as integer array in UTC. The format should be: [year,day_of_year,hours,minutes,seconds].
[out]freqFrequency of a tidal constituent
[out]phasePhase of a tidal constituent (Note 1)
Note
  1. The phases must be decreased by 90 degrees if the sum of the order and the species number is odd (as for the 2nd degree diurnals, and 3rd degree low frequency and semidiurnals). These phases may need further adjustment to allow for the spherical harmonic normalization used; e.g. for that used for the potential by Cartwright and Tayler, 180 degrees must be added for (species, order) = (1,2), (1,3), or (3,3).
  2. Status: Class 1 model
